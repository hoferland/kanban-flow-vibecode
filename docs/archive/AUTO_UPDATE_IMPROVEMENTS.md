## How the Auto-Update Works

### Current Configuration

**Updater is ENABLED** in `tauri.conf.json`:
```json
{
  "updater": {
    "active": true,
    "endpoints": [
      "https://github.com/YOUR_USERNAME/YOUR_REPO/releases/latest/download/latest.json"
    ],
    "dialog": false,
    "pubkey": "dW50cnVzdGVkIGNvbW1lbnQ6..."
  }
}
```

**Updater feature is included** in `Cargo.toml`:
```toml
tauri = { version = "1", features = ["updater", ...] }
```

### Update Flow

1. **App launches** → UpdateNotification component mounts
2. **Component calls** `checkUpdate()` from Tauri API
3. **Tauri fetches** `latest.json` from GitHub
4. **Compares versions** (current vs. latest)
5. **If update available** → Shows notification in UI
6. **User clicks "Update Now"** → Downloads and installs
7. **App restarts** automatically with new version

## ⚠️ CRITICAL ISSUE FOUND

**The auto-update is NOT working because:**

### Issue: No GitHub Releases Published

When testing the update endpoint, we get:
```bash
curl https://github.com/YOUR_USERNAME/YOUR_REPO/releases/latest/download/latest.json
# Returns: 404 Not Found
```

**Root Cause:** The repository either:
1. Has no releases published yet
2. The releases don't include the `latest.json` file
3. The repository might be private (API returns 404)

### How to Fix This

You need to create a GitHub release that includes the `latest.json` file. This file is automatically generated when you build with the `updater` target.

**Steps to fix:**

1. **Build with updater target:**
   ```bash
   cd packages/desktop
   TAURI_PRIVATE_KEY="$HOME/.tauri/flow-kanban.key" npm run tauri build
   ```

2. **Check that latest.json was created:**
   ```bash
   ls -la packages/desktop/src-tauri/target/release/bundle/dmg/latest.json
   ```

3. **Create a GitHub release (v0.2.1 if this is your first):**
   - Go to: https://github.com/YOUR_USERNAME/YOUR_REPO/releases/new
   - Tag: `v0.2.1`
   - Upload these files:
     - `Flow_0.2.1_aarch64.dmg` (or appropriate architecture)
     - `Flow_0.2.1_aarch64.dmg.tar.gz`
     - `Flow_0.2.1_aarch64.dmg.tar.gz.sig`
     - **`latest.json`** ← This is critical!

4. **Make repository public** (if it's currently private)
   - The updater needs public access to fetch `latest.json`
   - Or set up authentication (more complex)

### Why This Matters

Without `latest.json` on GitHub, the updater cannot:
- Check what version is available
- Compare versions
- Download updates
- Verify signatures

The error you're seeing is:
```
"Could not fetch a valid release JSON from the remote"
```

This literally means the `latest.json` file is not accessible at the configured endpoint.

## Other Potential Issues

### 1. Repository is Private
- Private repos return 404 to unauthenticated requests
- **Solution:** Make the repository public, or use GitHub tokens (complex)

### 2. Wrong Release Format
- Make sure you're using GitHub Releases, not just git tags
- The files must be attached as release assets

### 3. Wrong Architecture
- The endpoint includes architecture (e.g., `aarch64`)
- Make sure you're building for the correct architecture
